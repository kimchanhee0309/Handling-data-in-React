# 1. React로 데이터 다루기

<details>
  <summary>(1) 데이터 주도 UI란?</summary>

  * **한 줄 정의**: **데이터가 바뀌면 -> 화면이 저절로 바뀐다!**
  * **❌ 바닐라 JS - 직접 DOM을 만지는 방식**
    ```JavaScript
    const users = ['홍길동', '김철수', '이영희'];

    // 화면에 그리기
    const ul = document.createElement('ul');
    users.forEach(user => {
      const li = document.createElement('li');
      li.textContent - user;
      ul.appendChild(li);
    });
    document.body.appendChild(ul);

    // 사용자 한 명 추가하면?
    users.push('박민수');
    // 화면은 그대로! DOM을 또 조작해야 함
    ```

  * **✅ React - 데이터만 바꾸면 끝**
    ```JSX
    function UserList() {
      const [users, setUsers] = useState(['홍길동', '김철수', '이영희']);

      return (
        <ul>
          {users.map((user, i) => <li key={i}>{user}</li>))
        </ul>
      );
    }

    // 사용자 추가?
    setUsers([...users, '박민수']);
    // React가 알아서 화면 업데이트!
    ```
</details>

<details>
  <summary>(2) 선언형 vs 명령형(또는 절차형)</summary>

</details>

<details>
  <summary>(3) 데이터와 State</summary>

  * **정적 데이터 - 안 변하는 것**
    * 일반 변수로 선언하면 됨
      ```JSX
      function ProductList() {
        const categories = ['전자기기', '의류', '도서', '식품'];

        return (
          <ul>
            {categories.map((c, i) => <li key={i}>{c}</li>)}
          </ul>
        );
      }
      ```
       
  * **동적 데이터 - 변하는 것 -> State로!**
    * 사용자 입력, 버튼 클릭, API 응답 등으로 바뀌는 데이터는 반드시 `useState`로 관리
      ```JSX
      function TodoList() {
        const [todos, setTodos] = useState(['운동하기', '책 읽기']);

        function addTodo() {
          setTodos([...todos, '새 할일']);
        }

        return (
          <div>
            <ul>
              {todos.map((todo, i) => <li key={i}>{todo}</li>)}
            </ul>
            <button onClick={addTodo}>추가</button>
          </div>
        );
      }
      ``` 
</details>

<details>
  <summary>(4) 데이터 구조 설계하기</summary>

  * **Level 1. 단순 배열**
    * 간단한 목록에 적합
      ```JSX
      const fruits = ['사과, '바나나', '오렌지'];
      ```
      
  * **Level 2. 객체 배열⭐**
    * 항목마다 여러 정보를 담을 때
      ```JSX
      const products = [
        { id: 1, name: '노트북', price: 1200000, stock: 5 },
        { id: 2, name: '마우스', price: 30000, stock: 15 },
        { id: 3, name: '키보드', price: 80000, stock: 8 },
      ];
      ```
      * **id라는 고유 번호**가 있어야 특정 항목을 정확히 집어낼 수 있음
      * `map()`의 `key`값으로도 쓰임
      
  * **Level 3. 중첩 구조**
    * 복잡한 데이터를 표현할 때
      ```JSX
      const users = [
        {
          id: 1,
          name: '홍길동',
          profile: {
            age: 25,
            job: '개발자',
            skills: ['React', 'Node.js'],
          };
        };
      ];
      ```
</details>

<details>
  <summary>(5) 핵심 3줄 요약</summary>

  * 1. **React = 데이터 주도 UI**: 데이터만 바꿔라, 화면은 React가 그려준다.
  * 2. **변하는 데이터는 무조건 State**: `useState` 안 쓰면 화면 안 바뀜
  * 3. **실무에선 객체 배열 + id**: 거의 모든 상황에서 이 패턴을 씀

# 6. 배열 불변성 유지하기⭐

<details>
  <summary>(1) 불변성이란?</summary>

  * **왜 React는 불변성이 필수일까?**
    * React는 **"State가 바뀌었는지"를 참조값으로 비교함**
      ```JSX
      const items = [1, 2, 3];
      items.push(4);  // 원본 수정
      setItems(items);

      // React 속마음: "어? 같은 배열이네? (참조값이 동일)"
      //  -> 화면 업데이트 안 함
      ```
      ```JSX
      const items = [1, 2, 3];
      setItems([...items, 4]);  // 새 배열!

      // React 속마음: "오, 새 배열이네! 다시 그려야겟다!"
      // -> 화면 업데이트
      ```
      * **같은 메모리 주소를 가리키면 React는 "안 바뀌었다"고 판단함**
      * So, 항상 **새 배열/새 객체**를 만들어야 함
  
</details>

<details>
  <summary>(2) 원본을 바꾸는 위험한 메서드들</summary>

  |원본 수정(쓰지 말 것)|새 배열 반환(안전)|
  |---|---|
  |`push`(뒤에 추가)|`[...arr, item]`|
  |`unshift`(앞에 추가)|`[item, ...arr}`|
  |`pop`/`shift`(제거)|`filter`|
  |`splice`(중간 조작)|`filter`/`map`|
  |`sort`|`[...arr].sort()`|
  |`reveers`|`[...arr].reverse()`|
</details>

<details>
  <summary>(3) 추가하기</summary>

  * **뒤에 추가**
    ```JSX
    // X, 잘못된 방법
    function addItem() {
      items.push(newItem);  // 원본 수정
      setItems(items);      // 참조값 같음 -> 화면 안 바뀜
    }

    // O, 올바른 방법: 스프레드로 새 배열
    function addItem() {
      setItems([...items, newItem]);
    }
    ```
  * **앞에 추가
    ```JSX
    setItems([newItem, ...items]);
    ```
</details>

<details>

</details>

    ```JSX
      function updateSortOrder(newOrder) {
        setViewOptions({
          ...viewOptions,      // 기존 값들 유지
          sortOrder: newOrder, // sortOrder만 변경
          });
       }
    }
    ```
    
  * **주의: 스프레드 꼭 쓰기!**
    ```JSX
    // X, 이러면 filter, searchTerm이 날아감
    setViewOptions({ sortOrder: 'desc' });

    // O, 기존 값 유지하고 일부만 변경
    setViewOptions({ ...viewOptions, sortOrder: 'desc' });
    ```

  <summary>(3) ⭐️이 챕터의 핵심: "계산된 값은 State가 아니다"</summary>

  * **초보가 자주하는 실수**
    ```JSX
    function ProductList() {
      const [products, setProducts] useState([...]);
      const [filter, setFilter] = useState('all');
      const [filterProducts, setFilterProducts] = useState([]);  // 여기가 문제!!

      function handleFilterChange(newFilter) {
        setFilter(newFilter);
        // 필터 바꿀 때마다 filteredProducts도 수동으로 업데이트...
        setFilteredProducts(products.filter(p => p.category === newFilter));
      }
    }
    ```
    
  * **뭐가 문제?**
    * `filter`, `products`, `filteredProducts` 3개를 **계속 동기회**해야 함
    * 하나라도 까먹으면 **화면이 이상해짐**
    * 상태가 꼬여서 **디버깅 지옥**
      
  * **올바른 방법: 렌더링 시점에 계산**
    ```JSX
    function ProductList() {
      const [products, setProducts] = useState([...]);
      const [filter, setFilter] = useState('all');

      // State가 아니라, 렌더링될 때마다 "계산"
      const filteredProducts = products.filter(p => {
        if (filter === 'all') return true;
        return p.category === filter;
      });

      return (
        <div>
          {filteredProducts.map(product => (
            <div key={product.id}>{product.name}</div>
        </div>
      );
    }
    ```
